## 问题洞察

东方雨虹拥有 **3000余家品牌专卖店和10000余家终端网点，累计覆盖29万分销网点**，覆盖全国 **98%以上县域市场**，大量网点是县域"夫妻老婆店"——没有ERP、没有CRM、没有AI能力。2026年"3515计划"要在 **3年内助力507个县城各突破1000万业绩，总产值超50亿**，但县域经销商急需的不是复杂系统，而是"开箱即用"的AI赋能。

## 解决方案 · 飞书AI五大模块

方案围绕三大目标（渠道标准化管理体系、AI导购赋能平台、智能经营分析与预警中枢）构建五大模块：

| 模块 | 飞书AI能力 | 核心功能 | 项目目标 |
|------|-----------|----------|---------|
| 🔍 门店巡检 | 妙搭低代码 | 拍照→AI图像分析→异常识别→自动评分+改进建议 | 目标1-渠道标准化管理 |
| 🏛️ 渠道管理中枢 | 多维表格AI + aily | **原型已验证**：合规看板（区域/门店排名）/ 渠道洞察（区域/产品双维度）/ 跨门店库存看板 / 经营数据大屏<br>**二期规划**：智能分单 / 渠道拓展管理 / 履约风险监控 / 虹哥汇会员整合 | 目标1+目标3 |
| 💬 AI导购 | aily智能体 | 自然语言→意图识别→产品匹配→用量计算→施工步骤+费用；含话术训练、智能测评、效果图自动生成 | 目标2-AI导购赋能 |
| 📊 门店运营 | 多维表格AI字段 | 数据汇总→AI趋势/异常/机会识别→经营报告推送 | 目标3-智能经营分析 |
| 👥 客户维护 | AI标签+预测 | 画像分析→AI流失风险预测→个性化话术生成→定时跟进 | 目标3-智能经营分析 |

## 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 后端 | Python Flask | Web应用框架，22个API接口 |
| 前端 | Bootstrap 5 + Chart.js | 响应式UI + 数据可视化 |
| AI对话 | 飞书aily智能体 | 自然语言交互，产品推荐+施工指导 |
| 数据存储 | 飞书多维表格 | 轻量ERP，云端9表54+条记录+AI字段 |
| 低代码 | 飞书妙搭 | AI生成巡检应用，1343行代码 |
| API | 飞书开放平台 | tenant_access_token + Bitable + Aily |
| 部署 | GitHub Pages + 飞书云 | 无需自建服务器 |

> **数据口径说明：** 飞书多维表格云端存储9表54+条真实记录（含8个公式字段）；本地 `data/` 目录仅存放mock演示数据（产品/门店/巡检记录），供无飞书凭证时Demo运行使用。

## 成本透明度说明

本方案不声称"零成本"——飞书AI服务、大模型API调用、实施人力、培训、内容维护、运营支持均有真实费用。以下区分"软件成本"（AI平台费）与"落地总成本"（TCO，总拥有成本），完整透明拆解。

> **"两零一轻"说明：** "零硬件部署、零代码开发"指启动门槛（无需采购硬件、无需写代码），"轻量培训上手"指1天即可上手。**两零一轻指启动门槛，不等于零总成本**——真实落地需计入实施、培训、内容维护、运营支持等TCO科目。

- **1-2店**：aily免费版(400额度/月) + 多维表格基础版免费 + API约50元/店/年（仅API调用费）
- **3-20店**：飞书AI基础版9,900元/年均摊 + API约50元/店/年（仅API调用费）
- **100+店**：AI基础版9,900元/年均摊 + 多维表格专业版(按实际协作人数) + API约50元/店/年（仅API调用费）

> **注意：** 上述"50元/店/年"仅为API调用费，不等于总成本。真实落地还需计入实施、培训、内容维护、运营支持等TCO科目，见下表。
