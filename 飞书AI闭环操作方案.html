核心思路：飞书多维表格的「自动化」功能提供 9 种触发器和 15+ 种执行动作，支持「新增/修改记录满足条件时」触发，执行「新增记录」「修改记录」「发送消息」「创建任务」「发送HTTP请求」「AI生成文本」等操作。3个断点全部可以用这些原生能力解决，无需写任何代码。

2断点1：巡检不合格 → 自动创建整改任务

自动化流程 #1 巡检不合格自动触发整改

配置步骤

触发条件
触发器类型 新增/修改的记录满足条件时
数据表 巡检记录表

实际操作路径

  1. 打开多维表格 → 点击右上角 自动化 按钮
  2. 点击 + 创建自动化流程
  3. 左侧选择触发条件：新增/修改的记录满足条件时
  4. 选择数据表：巡检记录表；设置条件：总分 < 80
  5. 右侧第一个操作选 新增记录 → 选择「整改任务表」→ 配置字段映射
  6. 点击 + 添加第二个操作：发送飞书消息 → 配置接收人和消息卡片
  7. 点击 + 添加第三个操作：创建飞书任务 → 配置任务标题和负责人
  8. 点击 保存并启用

3断点2：AI推荐后无下单流程

这个断点需要两步解决：先创建一个「订单表单」让店员能快速下单，再配置自动化把订单同步到销售记录表。

操作步骤 创建订单表单视图 + 自动同步

步骤1：创建订单表单视图

  1. 在「订单记录表」中，点击左上角视图切换 → + 添加视图 → 选择 表单
  2. 命名为「快速下单表单」
  3. 勾选需要填写的字段：门店编号、客户姓名、产品编号、产品名称、购买数量、订单金额
  4. 设置「订单状态」默认值为「待发货」（在表单设置中固定）
  5. 点击 分享 → 获取表单链接，分享到门店店员群
使用方式：店员在AI导购中获得推荐产品后，直接打开表单链接，填写产品编号、数量、客户信息，30秒即可完成下单。订单自动写入「订单记录表」。

步骤2：配置自动化 - 订单同步到销售记录表

触发条件
触发器类型 添加新记录时
数据表 订单记录表
条件 订单编号 不为空（避免空白记录触发）
执行动作：新增记录到销售记录表
操作类型 新增记录
目标表 销售记录表
字段映射
门店编号 ← 引用订单记录「门店编号」
客户姓名 ← 引用订单记录「客户姓名」
产品名称 ← 引用订单记录「产品名称」
销售金额 ← 引用订单记录「订单金额」
销售日期 ← 引用订单记录「创建时间」
订单来源 ← 固定值「AI导购推荐」

步骤3（可选）：AI生成文本 - 智能推荐关联

执行动作：AI生成文本（订单摘要）
操作类型 AI 生成文本
指令
请根据以下订单信息，生成一段50字以内的客户购买摘要，用于客户画像分析：
客户：{客户姓名}，产品：{产品名称}，金额：{订单金额}元
输出到 订单记录表「购买摘要」字段（需提前新建该文本字段）
效果：店员在AI导购推荐后 → 打开表单30秒下单 → 订单自动同步到销售记录表 → AI自动生成购买摘要 → 客户跟进模块可直接读取。推荐转化率可追踪。

4断点3：销售/订单 → 库存自动扣减

自动化流程 #3 订单创建时自动扣减产品库存

当订单记录表中新增一条订单时，自动在产品库存表中找到对应产品，将「当前库存」减去购买数量。

触发条件
触发器类型 添加新记录时
数据表 订单记录表
条件 产品编号 不为空
执行动作 1：查找记录（定位产品库存）
操作类型 查找记录
目标表 产品库存表
筛选条件 产品编号 = 引用订单记录的「产品编号」
查找内容 当前库存、安全库存、产品编号
执行动作 2：修改记录（扣减库存）
操作类型 修改记录
目标表 产品库存表
修改范围 选择「之前步骤中对应的记录」（即上一步查找到的记录）
修改字段
当前库存 ← 查找记录的「当前库存」 - 订单记录的「购买数量」
（在字段值设置中，使用引用值相减的方式填入）
执行动作 3（可选）：库存预警通知
操作类型 条件判断 (If/Else) + 发送飞书消息
条件 如果扣减后「当前库存」< 「安全库存」
执行 发送消息到库存管理群：「产品{产品名称}库存不足！当前{当前库存}，安全库存{安全库存}，请尽快补货。」
效果：订单创建 → 自动查找产品 → 扣减库存 → 如果低于安全库存自动预警。全程零代码，全自动，避免超卖。

5aily智能体AI增强配置

除了3个断点的自动化修复，aily智能体可以在飞书内进一步深化AI能力，让闭环更加智能。

5.1 aily连接多维表格（知识库 + 数据源）

  1. 进入 aily.feishu.cn → 打开你的智能体
  2. 左侧菜单选择 知识库 → 添加知识库 → 选择 飞书多维表格
  3. 选择项目多维表格文件 → 勾选所有数据表
  4. aily会自动索引表格数据，支持自然语言查询（如"本月哪个区域合规率最低？"）

5.2 aily任务模式 - 定期经营分析

  1. 在aily中点击 任务 → 创建任务
  2. 设置定时触发：每周一上午9:00
  3. 任务指令示例：
aily任务指令模板
你是东方雨虹渠道运营分析师。请完成以下工作：

1. 从多维表格中读取本周所有门店的巡检记录、销售记录、整改任务完成情况
2. 计算各区域合规率排名、销售额排名、整改完成率
3. 识别本周合规率下降的门店和库存预警产品
4. 生成一份《本周渠道运营周报》，包含：
- 整体数据概览（巡检合格率、总销售额、平均客单价）
- 区域排名（合规率TOP3和BOTTOM3）
- 风险预警（库存不足产品、未完成整改门店）
- 下周建议（基于数据给出3条行动建议）
5. 将周报发送到「渠道运营管理」群

5.3 自动化流程中调用aily（AI生成文本）

飞书多维表格自动化流程内置了 AI生成文本 和 AI分类 动作，底层调用的就是飞书AI能力。可以在以下场景中使用：

场景触发AI动作输出
巡检报告生成 巡检记录提交后 AI生成文本 自动生成巡检分析摘要，写入巡检记录表「AI分析」字段
客户意向分类 客户表新增记录 AI分类 根据客户需求和购买记录，自动打上"高意向/中意向/低意向"标签
整改建议生成 整改任务创建后 AI生成文本 根据不合格维度自动生成整改建议，写入整改任务表

8注意事项

1. 字段名称必须完全一致：自动化流程中引用的字段名必须与多维表格中的字段名完全匹配（包括括号、空格）。建议先在表格中确认字段名，再配置自动化。
2. 自动化运行次数限制：飞书多维表格自动化流程有月度运行次数限制（根据企业版套餐不同，通常为每月数万次）。本项目7条自动化流程日常运行量不大，一般不会超限。
3. 「查找记录」+「修改记录」的串联：断点3的库存扣减需要先「查找记录」再「修改记录」，两个动作必须在同一个自动化流程中串联配置，第二个动作选择「之前步骤中对应的记录」才能正确关联。
4. 工作流 vs 自动化流程：飞书多维表格有两种流程引擎。「自动化流程」适合简单场景（1个触发器+1-3个动作）；「工作流」支持条件判断、循环、多分支等复杂逻辑，适合需要If/Else判断的场景（如库存预警）。在多维表格左下角可切换。
5. 测试方法：每条自动化配置完成后，可以在对应数据表中手动添加一条测试记录，观察是否触发了后续动作。也可以在自动化面板中点击「运行日志」查看执行记录和报错信息。
