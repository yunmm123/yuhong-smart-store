# 雨虹门店整改管家 - 妙搭应用提示词

## 使用方法

打开飞书妙搭 → 应用开发 → 将下方整段提示词粘贴到输入框 → 生成应用

## 妙搭输入框提示词（直接复制以下全部内容）

开发一个移动端门店整改管理应用「雨虹门店整改管家」，对接飞书多维表格数据源，主要供门店店长接收督导巡检后产生的整改任务、查看AI整改建议、提交整改结果。环境变量配置：FEISHU_APP_ID（飞书应用ID）、FEISHU_APP_SECRET（飞书应用密钥）、BITABLE_APP_TOKEN（多维表格app_token）、TABLE_RECTIFICATION（整改任务表table_id）、TABLE_STORE（门店信息表table_id），部署时在妙搭环境变量中配置。数据源整改任务表含20个字段：任务编号(Text如RT20260716001)、关联门店ID(Text存门店record_id)、关联巡检记录ID(Text存巡检记录ID)、门店编号(Text从门店信息表同步)、门店名称(Text)、区域(SingleSelect华东/华南/华北/华中/西南/西北/东北)、巡检时间(DateTime)、巡检总分(Number)、低分维度(Text)、AI整改建议(Text)、整改要求(Text)、截止日期(DateTime)、任务状态(SingleSelect待处理/已接收/整改中/已完成/已超期)、店长姓名(Text)、店长手机号(Phone)、接收时间(DateTime)、完成时间(DateTime)、整改说明(Text)、整改照片(Attachment)、创建时间(DateTime)。门店信息表含门店编号(Text)、门店名称(Text)、区域(SingleSelect)、店长姓名(Text)、店长手机号(Phone)等字段。应用需包含4个页面：第一个页面「我的任务」是首页，应用启动时通过飞书JS SDK自动获取当前登录用户的手机号，从整改任务表中筛选店长手机号等于当前用户手机号的记录，实现按店长身份自动匹配门店并精准隔离数据，下方是任务卡片列表，每张卡片显示任务编号、任务状态标签（待处理红色/已接收橙色/整改中蓝色/已完成绿色/已超期灰色）、巡检总分、低分维度、截止日期和剩余天数超期标红、AI整改建议摘要点击展开，顶部有全部/待处理/整改中/已完成/已超期筛选标签，超期任务排最前，列表加载时前端自动判断截止日期超过当前时间且状态不是已完成的任务显示为已超期状态；第二个页面「任务详情」点击任务卡片进入，展示任务编号、状态、巡检信息（巡检时间、总分、低分维度）、AI整改建议完整内容紫色边框突出、整改要求、截止日期倒计时，底部根据任务状态显示操作按钮（待处理显示「接收任务」按钮点击后状态变为已接收并记录接收时间，已接收显示「开始整改」按钮点击后状态变为整改中，整改中显示「提交整改」按钮跳转到提交页面，已完成显示完成时间和整改说明只读）；第三个页面「提交整改」从任务详情进入，顶部显示任务编号和门店名称，包含整改说明多行文本输入框必填至少20字、整改照片上传支持多张拍照或相册选择至少1张，底部「提交整改」按钮点击后将整改说明和照片写入多维表格整改说明和整改照片字段、任务状态更新为已完成、完成时间写入当前时间、弹出成功提示后返回任务列表；第四个页面「本店看板」仅显示当前门店的统计，包含待处理/整改中/已完成/超期四个数字卡片、整改完成率圆环图、近30天整改趋势折线图、历史已完成整改记录列表每条显示任务编号、完成日期、整改说明摘要。数据隔离要求：所有数据按店长手机号自动过滤，店长只能看到自己门店的整改任务，看不到其他门店数据和巡检评分明细。UI风格：移动端优先，主色调雨虹红#E60012，背景#f5f5f5，白色卡片圆角12px，底部固定Tab栏4个Tab。飞书API调用方式：获取token用POST https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal传app_id和app_secret，读取整改任务表记录用GET https://open.feishu.cn/open-apis/bitable/v1/apps/{BITABLE_APP_TOKEN}/tables/{TABLE_RECTIFICATION}/records支持filter参数按店长手机号过滤，更新记录用PATCH同路径下/records/{record_id}传fields对象，上传附件先POST https://open.feishu.cn/open-apis/drive/v1/medias/upload_all获取file_token再PATCH写入整改照片字段。应用部署后设置可见范围为租户内可见。
