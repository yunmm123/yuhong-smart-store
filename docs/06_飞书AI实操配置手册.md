## 第一部分：创建飞书应用获取 API 凭证（30 分钟）

飞书自建应用是访问 aily 智能体、多维表格、妙搭等能力的"通行证"。本部分将创建一个企业自建应用，并获取 `App ID` 与 `App Secret`。

### 1.1 创建飞书应用

1. 浏览器打开 **飞书开放平台**：https://open.feishu.cn/app
2. 使用你在"前置准备"中注册的飞书账号登录。
3. 点击页面右上角（或中央）的 **"创建企业自建应用"** 按钮。
4. 在弹出的表单中填写：
   - **应用名称**：`雨虹渠道智慧运营助手`
   - **应用描述**：`东方雨虹渠道智慧运营项目，基于飞书AI生态构建的建材渠道智慧运营一体化解决方案`
   - **应用图标**：可上传项目 `static/` 目录下的雨虹 logo，或使用默认图标。
   - **应用分组**：可留空或选择"内部应用"。
5. 点击 **"创建"**，等待 3-5 秒跳转到应用详情页。

> 提示：若提示"无权限创建应用"，说明你当前账号不是团队管理员。请在飞书管理后台（https://feishu.cn/admin）把你的账号设为管理员，或让管理员在"工作台 → 应用管理 → 开发者后台"中为你开通开发权限。

### 1.2 获取 App ID 与 App Secret

1. 在应用详情页左侧菜单点击 **"凭证与基础信息"**。
2. 你会看到两个关键值：
   - **App ID**（格式形如 `cli_a1b2c3d4e5f6g7h8`）
   - **App Secret**（一串 32 位字符串，仅显示一次，请立即复制保存）
3. 将这两个值暂时记到记事本，稍后会填入项目配置。
   - 如果 App Secret 没有复制下来，可以点击"重置 App Secret"重新生成。

> 安全提示：App Secret 等同于应用密码，切勿提交到公开 Git 仓库或截图发到群聊。项目已通过 `.gitignore` 忽略 `.env` 文件，请只把凭证放在 `.env` 中。

### 1.3 配置应用权限

飞书自建应用默认没有任何权限，需要按本项目的实际用途逐项开启。

1. 在左侧菜单点击 **"权限管理"**。
2. 在搜索框中依次搜索并 **勾选开通** 以下权限（开通后状态变为"已开通"）：

   | 权限标识 | 权限名称 | 用途 |
   |----------|----------|------|
   | `bitable:app` | 查看、评论、编辑和管理多维表格 | 读写门店运营管理表 |
   | `bitable:app:readonly` | 查看多维表格 | 仅读取场景（可选） |
   | `aily:aily` | 使用 aily 智能体 | 调用 AI 导购对话 |
   | `aily:aily:readonly` | 查看 aily 智能体信息 | 读取智能体配置（可选） |
   | `contact:user.base:readonly` | 获取用户基础信息 | 获取巡检人/客户身份 |
   | `contact:user.id:readonly` | 获取用户 ID | 用于 aily 会话 user_id 传参 |

3. 开通权限后，部分权限会提示"需要管理员审批"。点击页面顶部的 **"批量开通"** → **"提交管理员审批"**。
   - 如果你自己就是管理员，进入飞书管理后台 **"工作台 → 应用管理 → 待审批"** 中通过即可，通常秒批。

### 1.4 配置应用可用范围

权限开通后，还需要把应用发布给团队成员使用。

1. 左侧菜单点击 **"版本管理与发布"**。
2. 点击 **"创建版本"**，填写：
   - **版本号**：`1.0.0`
   - **版本说明**：`首版上线，包含多维表格读写、aily智能体调用能力`
   - **可用范围**：选择"全部成员"或指定自己的账号（Demo 阶段指定自己即可）
3. 点击 **"保存"** → **"申请发布"**。
4. 企业内部应用通常 **秒过审核**，状态会变为"已发布"。
5. 发布成功后，在飞书客户端"工作台"中即可看到"雨虹渠道智慧运营助手"应用图标。

> 检查点：到这里你应该已经拿到 `App ID` 和 `App Secret` 两个值。如果还没拿到，回到 1.2 步重新获取。

## 第二部分：创建 aily 智能体（45 分钟）

aily（AI LackeY）是飞书的智能体搭建平台，无需写代码即可配置出一个具备知识库、术语库、人设的 AI 助手。本部分将创建"雨虹 AI 导购助手"。

### 2.1 进入 aily 开发后台

1. 浏览器打开 **aily 智能体平台**：https://aily.feishu.cn
2. 使用与第一部分相同的飞书账号登录。
3. 点击 **"创建智能体"** → 选择 **"自定义智能体"**（不要选模板，我们要从零配置）。
4. 在弹窗中填写：
   - **智能体名称**：`雨虹AI导购助手`
   - **智能体描述**：`东方雨虹建材产品导购专家，提供防水材料选型、施工方案、用量计算等专业咨询`
   - **头像**：上传雨虹 logo 或使用默认 AI 头像
5. 点击 **"确定"**，进入智能体配置主界面。

> 界面说明：aily 配置主界面左侧是"编排"菜单，包含"人设与回复逻辑""知识空间""术语库""工具""发布"等模块，本部分会依次配置前四个。

### 2.2 配置智能体人设

人设（System Prompt）决定了 aily 的"性格"和"专业边界"。一个高质量的人设能让回复更稳定、更专业。

1. 左侧菜单点击 **"编排" → "人设与回复逻辑"**。
2. 在 **"系统提示词"** 文本框中，完整粘贴以下内容：

```
你是"雨虹AI导购助手"，由东方雨虹与飞书AI联合打造的建材防水领域专业导购智能体。

【你的身份】
- 姓名：小虹
- 角色：东方雨虹认证建材导购专家，拥有10年防水行业经验
- 服务对象：东方雨虹零售门店的终端消费者、装修师傅、门店店员

【你的能力】
1. 产品知识问答：精通雨虹全线产品（防水涂料、防水卷材、瓷砖胶、密封胶、美缝剂、修缮材料等）的参数、用途、价格、施工工艺
2. 场景化方案推荐：根据用户描述的漏水部位（卫生间/厨房/阳台/屋顶/外墙/地下室）给出完整的材料组合方案
3. 用量计算：根据施工面积和产品每平米用量，精确计算所需材料数量和预估费用
4. 施工指导：输出标准化的施工步骤、注意事项、施工难度评级
5. 售前售后咨询：解答产品对比、质保、采购渠道等问题

【回复规范】
- 开头先简短复述并确认用户需求，让用户感到被理解
- 推荐产品时，必须给出：产品全称、适用场景、预估用量、预估费用、施工难度
- 涉及施工时，必须给出"施工步骤"和"注意事项"两个结构化模块
- 价格一律使用人民币（¥），单位与产品包装对应（桶/卷/袋/支/组）
- 遇到不确定的问题（如具体漏点），主动追问 1-2 个关键问题，不要臆测
- 不夸大宣传，不做医学/结构安全承诺，涉及结构问题建议联系专业工程师

【语气风格】
- 专业、亲切、耐心，像一个经验丰富的师傅在身边指导
- 适度使用emoji（如🏠💧✅）增强可读性，但不滥用
- 回复控制在200-400字，结构清晰，多用分点

【知识边界】
- 仅回答防水、建材、装修相关问题
- 被问到竞品（如德高、科顺、西卡）时，客观介绍不贬低，重点强调雨虹产品差异化优势
- 被问到与建材无关的问题（如天气、新闻、娱乐），礼貌引导回防水话题

【示例对话】
用户：卫生间漏水怎么处理？
小虹：您好！卫生间漏水是家装最常见的烦恼之一💧。为了给您最准确的方案，想先确认一下：
1. 漏水点是地面、墙面还是管根周围？
2. 是新房装修预防，还是老房已经出现渗漏？
如果是老房渗漏，我推荐以下组合方案：
- 【雨虹JS复合防水涂料】×涂刷2-3遍，适合墙面和地面，¥320/桶(20kg)，每平米用量约2.5kg
- 【雨虹硅酮密封胶】×处理管根、阴角等细节部位，¥38/支(300ml)
请告诉我卫生间面积，我帮您精确计算用量和费用💰
```

3. 在 **"开场白"** 处填写：
```
您好！我是雨虹AI导购助手小虹🏠，专注防水建材选型与施工方案咨询。您可以这样问我：
• "卫生间漏水怎么处理？"
• "30平米屋顶用SBS卷材需要多少？"
• "JS防水涂料怎么施工？"
请问您今天想咨询哪方面的问题？
```

4. 在 **"推荐问题"**（部分版本叫"引导问题"）中添加 3 条：
   - `卫生间漏水怎么处理？`
   - `我家屋顶30平米需要多少防水材料？`
   - `SBS卷材怎么施工？`

5. 点击页面右上角 **"保存"**。

### 2.3 创建知识空间并导入产品知识

知识空间是 aily 检索外部资料的"图书馆"。本项目有 8 款雨虹产品的详细参数，需要导入进来。

#### 2.3.1 新建知识空间

1. 左侧菜单点击 **"知识空间"** → **"新建知识空间"**。
2. 填写：
   - **名称**：`雨虹产品知识库`
   - **描述**：`东方雨虹全线产品参数、施工工艺、注意事项`
3. 点击 **"创建"**。

#### 2.3.2 准备并上传产品知识文件

项目 `data/products.json` 包含 8 款产品。aily 对结构化表格支持较好，推荐转换为 Excel 后上传，也可手工录入飞书文档。

**方式 A：转换 JSON 为 Excel 上传（推荐，数据最全）**

1. 在本地项目根目录新建一个临时脚本 `convert_products.py`（可放在 `/data/user/work` 或项目根目录）：

```python
import json
import pandas as pd

with open('data/products.json', 'r', encoding='utf-8') as f:
    products = json.load(f)

# 将列表型字段拼接为字符串，便于表格展示
rows = []
for p in products:
    rows.append({
        '产品编号': p['id'],
        '产品名称': p['name'],
        '品类': p['category'],
        '产品描述': p['description'],
        '价格(元)': p['price'],
        '单位': p['unit'],
        '每平米用量': p['usage_per_sqm'],
        '评分': p['rating'],
        '关键词': '、'.join(p['keywords']),
        '适用场景': '、'.join(p['applications']),
        '施工步骤': '\n'.join([f'{i+1}. {s}' for i, s in enumerate(p['construction_steps'])]),
        '注意事项': '\n'.join([f'- {c}' for c in p['precautions']]),
        '施工时长': p['estimated_time'],
        '施工难度': p['difficulty'],
    })

df = pd.DataFrame(rows)
df.to_excel('products.xlsx', index=False)
print('已生成 products.xlsx')
```

2. 运行 `pip install pandas openpyxl && python convert_products.py`，得到 `products.xlsx`。
3. 回到 aily 知识空间，点击 **"添加知识" → "本地文件"**，上传 `products.xlsx`。
4. 上传完成后，aily 会自动解析并建立索引，状态变为"已就绪"。

**方式 B：飞书文档手工录入（适合演示效果，更直观）**

1. 在知识空间点击 **"添加知识" → "飞书文档" → "新建文档"**，命名 `雨虹产品手册`。
2. 为每款产品建立一个二级标题，结构如下（以 JS 涂料为例）：

```
## 雨虹JS复合防水涂料（P001）
- 品类：防水涂料
- 价格：320元/桶(20kg)
- 每平米用量：2.5kg
- 适用场景：卫生间、厨房、阳台、室内
- 施工步骤：
  1. 基层处理：清理表面，确保平整、无油污、无松动
  2. 配料：按液料:粉料=1:2.5比例混合搅拌均匀
  3. 第一遍涂刷：用滚筒均匀涂刷，厚度约0.5mm
  4. 等待表干（约4小时）后涂刷第二遍
  5. 涂刷第三遍，总厚度达到1.5mm以上
  6. 24小时后进行闭水试验
- 注意事项：施工温度5℃-35℃；基层含水率不超过20%；每遍涂刷方向交叉；涂膜未干前禁止踩踏
- 施工难度：中等
```

3. 重复上述步骤，把 8 款产品（P001-P008）全部录入。参考项目 `data/products.json` 的字段。

#### 2.3.3 关联知识空间到智能体

1. 回到智能体配置主界面，左侧 **"编排" → "知识空间"**。
2. 勾选刚创建的 **"雨虹产品知识库"**，并开启 **"知识空间检索"** 开关。
3. 检索策略建议选择 **"语义检索 + 关键词检索"**（混合检索），召回更准。
4. 设置 **"Top K"** 为 `3`（每次对话最多引用 3 条知识）。
5. 点击 **"保存"**。

### 2.4 配置术语库

术语库让 aily 在回复时正确解释建材专业名词，避免"望文生义"。本项目涉及大量防水专业术语，必须配置。

1. 在知识空间页面点击 **"术语库"** 标签 → **"新建术语库"**，命名 `雨虹建材术语库`。
2. 逐条添加以下术语（点击"添加术语"，填入"术语"和"解释"）：

| 序号 | 术语 | 解释 |
|------|------|------|
| 1 | SBS | 苯乙烯-丁二烯-苯乙烯嵌段共聚物，一种热塑性弹性体，用于改性沥青防水卷材，赋予卷材良好的低温柔性和延伸性 |
| 2 | JS防水涂料 | 聚合物水泥防水涂料，J=聚合物、S=水泥，以聚合物乳液和水泥为主要原料的双组份防水涂料，柔韧性好 |
| 3 | 闭水试验 | 防水施工完成后进行的蓄水检验，蓄水深度不低于20mm，蓄水时间不少于24小时，观察楼下有无渗漏 |
| 4 | 一次涂刷 | 单遍涂刷防水涂料的厚度，通常0.5mm左右，需多遍涂刷达到设计总厚度 |
| 5 | 表干 | 涂膜表面已干燥结膜，手指轻触不粘，但内部未完全固化，通常需4-6小时 |
| 6 | 实干 | 涂膜内部完全固化干燥，可进行下道工序，通常需24-48小时 |
| 7 | 冷底子油 | 又称底油，由沥青加溶剂稀释而成，涂刷在基层上增强卷材与基层的粘结力 |
| 8 | 热熔法 | 用喷灯或火焰加热器烘烤防水卷材底面沥青，使其熔融后与基层粘结的施工方法 |
| 9 | 搭接宽度 | 相邻两幅防水卷材重叠粘结的宽度，SBS卷材长边搭接不少于80mm，短边不少于100mm |
| 10 | 聚氨酯 | 一种高分子防水材料，分为单组份和双组份，延伸率高、耐候性好，常用于屋面和地下室 |
| 11 | 玻纤布 | 玻璃纤维增强布，铺设在防水涂料层中起增强作用，提高涂膜抗拉强度 |
| 12 | 管根 | 管道穿越楼板或墙体的根部位置，是渗漏高发部位，需用密封胶加强处理 |
| 13 | 阴角 | 墙面与地面、墙面与墙面交接的内凹部位，防水需做圆弧处理并加铺增强层 |
| 14 | 美缝 | 在瓷砖缝隙中填充美缝剂，起防水、防霉、装饰作用的工序 |
| 15 | 蓄水试验 | 同闭水试验，部分地区称谓不同 |

3. 添加完成后，回到智能体 **"编排" → "术语库"**，关联 `雨虹建材术语库`，点击"保存"。

### 3.5 获取多维表格 app_token 和各 table_id

1. **获取 app_token（多维表格令牌）**：

2. **获取各 table_id（数据表 ID）**：
   - 切换到 `门店信息` 表，URL 会变为 `...?table=tblAAAA`，`tblAAAA` 就是该表的 table_id。
   - 依次切换到巡检记录、产品库存、客户信息、销售记录，记录每个表对应的 `tbl...` 值。

3. 对照项目 `config.py` 的字段映射：

   | config.py 字段 | 对应数据表 | 你获取的值 |
   |----------------|-----------|-----------|
   | `BITABLE_APP_TOKEN` | 多维表格整体 | `xxxxxxxxxx` |
   | `BITABLE_STORE_TABLE_ID` | 门店信息 | `tbl...` |
   | `BITABLE_INSPECTION_TABLE_ID` | 巡检记录 | `tbl...` |
   | `BITABLE_CUSTOMER_TABLE_ID` | 客户信息 | `tbl...` |
   | `BITABLE_SALES_TABLE_ID` | 销售记录 | `tbl...` |

> 检查点：本部分结束后，记事本应有 7 个值：FEISHU_APP_ID、FEISHU_APP_SECRET、AILY_APP_ID、BITABLE_APP_TOKEN，以及 4 个 TABLE_ID（产品库存表本项目代码暂未单独配置，可与门店表共用或忽略）。

### 5.1 更新项目配置

1. 在项目根目录复制环境变量模板：


## 附录：常见问题

汇总 10 个最常见的配置问题及解决方案，按出现频率排序。

### Q1：调用 aily 接口报错"获取 token 失败：code 99991663"
**原因**：App ID 或 App Secret 填错，或应用未发布。
**解决**：
1. 回到 https://open.feishu.cn/app 核对 App ID / Secret。
2. 确认"版本管理与发布"中应用状态为"已发布"，而非"开发中"。
3. 确认 `.env` 中没有多余空格或换行。

### Q2：多维表格 API 报"permission denied"（403）
**原因**：应用未开通 `bitable:app` 权限，或多维表格未授权给应用。
**解决**：
1. 开放平台 → 权限管理，确认 `bitable:app` 已开通。
2. 打开多维表格 → 右上角"…" → "添加协作者" → 搜索应用名"雨虹渠道智慧运营助手" → 设为"可编辑"。
3. 重新发布应用版本。

### Q3：aily 回复没有用到知识库，产品价格全是错的
**原因**：知识空间未关联，或文件还在解析中。
**解决**：
1. aily 编排 → 知识空间，确认"雨虹产品知识库"已勾选且"知识空间检索"开关已开。
2. 进入知识空间列表，确认 `products.xlsx` 状态为"已就绪"而非"解析中"。
3. 若仍无效，把检索策略改为"混合检索"，Top K 调到 5 重试。

### Q4：妙搭提交巡检单后，多维表格没有新增记录
**原因**：字段映射不一致或目标表选错。
**解决**：
1. 妙搭 → 数据 → 数据模型，确认关联的是"巡检记录"表（不是门店信息表）。
2. 逐一核对表单组件名与多维表格列名是否完全相同（注意全角/半角括号、空格）。
3. 多维表格中的公式字段（总分、巡检结果）不要在映射里出现。

### Q5：多维表格 AI 字段生成内容为空或报错
**原因**：提示词中引用的字段名与实际列名不一致，或所选模型未开通。
**解决**：
1. 编辑 AI 字段，删除手写的 `{字段名}`，改用编辑器的"插入字段"按钮重新选择。
2. 切换模型：豆包 → GPT-4o（需企业版开通）→ 通义千问。
3. 检查该行是否所有被引用字段都有值，空值会导致生成失败。

### Q6：Flask 网站显示"未检测到飞书API凭证，使用模拟数据"
**原因**：`.env` 未被加载，或环境变量名拼错。
**解决**：
1. 确认项目根目录有 `.env` 文件（不是 `.env.example`）。
2. 检查 `app.py` 是否 `from dotenv import load_dotenv; load_dotenv()`，没有就手动加。
3. 命令行执行 `export $(grep -v '^#' .env | xargs)` 强制注入环境变量后再启动。

### Q7：飞书应用审核一直不通过
**原因**：企业内部应用一般秒过，卡住通常是可用范围或权限配置问题。
**解决**：
1. 版本管理 → 检查"可用范围"是否包含自己。
2. 权限管理 → 确认所有"已申请"权限都显示"已开通"。
3. 联系团队管理员在飞书管理后台审批。

### Q8：术语库添加后 aily 回复仍用错术语
**原因**：术语库未关联到智能体。
**解决**：
1. aily 编排 → 术语库，确认"雨虹建材术语库"已被勾选。
2. 术语库的"生效范围"设为"全部对话"。
3. 在调试预览中重新提问验证。

### Q9：录屏时飞书界面文字模糊
**原因**：屏幕缩放比例过高或录制分辨率不足。
**解决**：
1. 系统显示设置 → 缩放调到 100%。
2. OBS 输出分辨率设为 1920×1080，码率 2500kbps 以上。
3. 浏览器缩放调到 100%，关闭"高 DPI 缩放覆盖"。

### Q10：视频上传 B 站后画质被压缩严重
**原因**：原视频码率过低或 B 站转码二次压缩。
**解决**：
1. 原视频码率不低于 5000kbps，分辨率 1080p。
2. B 站上传时勾选"投稿画质：1080P 高码率"（需 UP 主等级足够）。
3. 备用方案：同时上传到腾讯视频，提供双链接。
